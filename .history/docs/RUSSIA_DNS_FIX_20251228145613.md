# Решение проблем с DNS для пользователей из РФ

## Проблема

В России DNS запросы могут блокироваться на уровне провайдера, что приводит к тому, что сайты не открываются даже через VPN.

## Решения

### Вариант 1: DNS через VPN туннель (Рекомендуется)

Использовать DNS сервер который работает через VPN туннель. Можно использовать:
- Cloudflare DNS через VPN: `1.1.1.1, 1.0.0.1`
- Google DNS через VPN: `8.8.8.8, 8.8.4.4`
- AdGuard DNS: `94.140.14.14, 94.140.15.15`
- Quad9: `9.9.9.9`

**Но важно:** DNS запросы должны идти через VPN, а не напрямую через интернет провайдера.

### Вариант 2: Собственный DNS сервер на VPN сервере

Установить DNS сервер (например, Unbound или dnsmasq) на VPN сервере и использовать его IP как DNS в конфигурации клиентов.

### Вариант 3: DNS over HTTPS (DoH)

Использовать DoH провайдеров, которые обходят блокировки.

## Быстрое решение

Изменить DNS в конфигурации WireGuard сервера на более надежные варианты, которые лучше работают в России.
